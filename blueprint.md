# 마음의 온도 (Temperature of the Heart) - Blueprint (Phase 3: 유리병 편지)

## 1. 프로젝트 컨셉 재정의: "유리병 편지" 모델

기존의 개방된 커뮤니티 게시판 모델에서, 사용자의 고민을 '유리병 편지'에 담아 바다에 띄우면 불특정 누군가에게 무작위로 전달되는, 보다 개인적이고 감성적인 상호작용 모델로 전환합니다. 여기에 게임화 요소를 도입하여 사용자의 지속적인 참여를 유도합니다.

**핵심 사용자 경험:**
1.  **고민 띄우기:** 사용자는 하루에 한 번 주어지는 '고민권'을 사용하여 자신의 고민을 유리병에 담아 바다로 띄워 보냅니다.
2.  **답장 받기:** 내가 보낸 유리병을 주운 다른 사용자가 답장을 써서 보내줍니다.
3.  **답장하기:** 나 또한 다른 사람의 유리병을 줍고, 그들의 고민에 대한 답장을 써줄 수 있습니다.
4.  **보상 및 성장:** 나의 답장이 채택되면 보상을 받고, 다른 사람들의 공감을 얻어 '마음의 온도'가 올라가는 경험을 합니다.

## 2. 주요 기능 및 게임화 요소

- **유리병 편지 (핵심 기능):**
    - 작성된 고민은 '게시'되는 것이 아니라, 특정 사용자에게 '전달'됩니다. (백엔드 부재로 프로토타입에서는 시뮬레이션)
- **고민권 시스템:**
    - 매일 자정, 모든 사용자에게 1개의 '고민권'이 지급됩니다.
    - 고민을 작성하려면 반드시 고민권이 필요합니다.
- **답장 채택 및 보상:**
    - 내 고민에 달린 여러 답장 중 가장 마음에 드는 답장을 '채택'할 수 있습니다.
    - 내 답장이 채택된 사용자는 보상을 받습니다.
    - **보상 종류:** 추가 고민권, 답장을 꾸밀 수 있는 '편지지 스킨'.
- **마음의 온도 (점수 시스템):**
    - 모든 유저는 '마음의 온도'라는 수치를 가집니다.
    - 다른 사용자로부터 공감을 얻을 때마다 온도가 올라갑니다.
    - **온도 상승 조건:** 내 고민에 답장을 받을 때, 내 답장이 채택될 때 등.

## 3. 기술적 한계 및 프로토타이핑 전략

**[중요] 기술적 한계:**
본 프로젝트는 **백엔드 서버 없이** 프론트엔드(HTML, CSS, JavaScript)만으로 구현됩니다. 따라서, 사용자 간의 실시간 데이터 교환(고민 전달, 답장 전송)은 불가능합니다.

**시뮬레이션 전략:**
이러한 한계를 극복하고 사용자 경험의 핵심을 프로토타입으로 구현하기 위해 다음 전략을 사용합니다.
- **모든 데이터는 `localStorage`에 저장:** 사용자 정보, 고민, 답장 등 모든 데이터는 사용자 개인의 브라우저에만 저장됩니다.
- **'다른 사람의 고민' 풀(Pool) 운영:** `localStorage`에 미리 여러 개의 '다른 사람의 고민' 데이터를 저장해 둡니다. 사용자가 '유리병 줍기'를 시도하면, 이 풀에서 무작위로 하나를 꺼내 보여줍니다.
- **'나의 고민' 관리:** 내가 작성한 고민은 `localStorage`에 저장되며, 답장 또한 시뮬레이션 로직(예: 일정 시간 후 자동으로 생성)을 통해 '받은 것처럼' 보여줄 수 있습니다.

## 4. 개발 계획 (Phase 3)

### 1단계: 데이터 구조 및 UI 재설계
- **데이터 구조 개편 (`main.js`):**
    - **User Object:** `temperature`, `vouchers`, `lastLoginDate`, `unlockedSkins` 등을 포함하는 사용자 객체를 `localStorage`에 관리.
    - **Worry Object:** `status` ('sent', 'replied'), `recipient` (시뮬레이션용) 등의 상태를 추가.
- **HTML/CSS 리팩토링 (`index.html`, `style.css`):**
    - 메인 화면을 '바다' 테마로 전면 개편.
    - 네비게이션을 "바다(홈)", "받은 편지함", "보낸 편지함"으로 변경.
    - '편지지 스킨'을 적용할 수 있는 답장 작성 UI 디자인.
    - '마음의 온도', '고민권 개수'를 표시하는 UI 요소 추가.

### 2단계: 핵심 메커니즘 시뮬레이션 구현
- **고민권 시스템 (`main.js`):**
    - 앱 초기화 시 `lastLoginDate`를 확인하여 하루가 지났으면 고민권을 1개 충전하는 로직 구현.
- **유리병 띄우기/줍기 (`main.js`):**
    - '고민 띄우기' 시 고민권을 차감하고, `Worry Object`를 생성하여 `localStorage`에 '보낸 편지함'으로 저장.
    - '유리병 줍기' 시, '다른 사람의 고민' 풀에서 아직 답장하지 않은 고민을 무작위로 제시.
- **답장 및 채택 (`main.js`):**
    - 다른 사람의 고민에 답장하는 기능 구현.
    - '보낸 편지함'에서 내가 받은 답장을 보고 '채택'하는 기능 구현.
- **보상 시스템 (`main.js`):**
    - (시뮬레이션) 내가 쓴 답장이 '채택'되었다고 가정하고(예: 랜덤 로직), 사용자에게 '추가 고민권'이나 '편지지 스킨'을 지급하는 로직 구현.
- **마음의 온도 (`main.js`):**
    - 답장을 받거나, 내 답장이 채택될 때 '마음의 온도' 점수가 오르고 UI에 반영되는 로직 구현.

- **다음 단계:** 새로운 컨셉에 대한 사용자 동의 확인 후, 데이터 구조 개편부터 개발 시작.
