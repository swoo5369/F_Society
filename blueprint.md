# 마음의 온도 (Temperature of the Heart) - Blueprint

## 1. 프로젝트 개요

"마음의 온도"는 익명의 사용자들이 자신의 고민을 털어놓고, 다른 사용자나 AI로부터 따뜻한 위로의 답장을 받을 수 있는 감성 기반 커뮤니티 웹 애플리케이션입니다. 거친 언어가 오가는 기존 커뮤니티와 달리, 따뜻하고 부드러운 소통을 지향합니다. '새벽 감성'을 자극하는 디자인을 통해 사용자들이 가장 감성적인 시간대에 위로를 얻을 수 있는 공간을 제공하는 것을 목표로 합니다.

## 2. 핵심 기능 및 디자인

### 주요 기능
- **익명 고민 작성:** 사용자는 익명으로 자신의 고민이나 이야기를 자유롭게 작성할 수 있습니다.
- **위로의 답장:** 작성된 고민에 대해 다른 사용자나 AI가 '위로의 답장'을 남깁니다.
- **AI의 따뜻한 답변:** AI가 생성한 따뜻하고 희망적인 메시지를 제공합니다. (초기 버전에서는 미리 정의된 메시지 사용)
- **답변 이미지 캡처:** 마음에 드는 위로의 답변을 이미지 파일로 저장하여 소셜 미디어 등에 공유할 수 있는 기능을 제공합니다.
- **내 게시물 목록:** 사용자가 작성한 모든 고민과 거기에 달린 답장들을 한눈에 볼 수 있는 페이지를 제공합니다.
- **답장 채택:** 자신의 고민에 달린 여러 답장 중 가장 마음에 드는 답장을 '채택'하여 특별하게 표시할 수 있습니다.
- **커뮤니티 참여:** 다른 사용자들이 남긴 고민에 직접 위로의 답장을 작성하며 커뮤니티에 참여할 수 있습니다.

### 디자인 컨셉: "새벽 감성"
- **Color Palette:** 전체적으로 어두운 네이비, 보라색 계열을 메인으로 사용하고, 포인트 컬러로 따뜻한 느낌의 오렌지, 옐로우, 핑크를 사용하여 새벽의 차분함과 따스함을 동시에 표현합니다.
- **Typography:** 부드러운 느낌의 산세리프 폰트를 사용하여 가독성과 감성적인 분위기를 모두 만족시킵니다.
- **Layout:** 여백을 충분히 활용한 미니멀리즘 디자인으로 사용자가 내용에 집중할 수 있도록 합니다.
- **Visuals:** 밤하늘, 별, 부드러운 빛과 같은 감성적인 일러스트나 배경 요소를 활용합니다.

## 3. 기술 계획 (Phase 2)

### 데이터 구조
- 모든 데이터(고민, 답장)는 객체 형태로 관리되며, `localStorage`를 사용하여 브라우저에 저장됩니다.
- **Worry Object:**
    - `id`: (string) Unique ID
    - `text`: (string) 고민 내용
    - `timestamp`: (Date) 작성 시간
    - `isMe`: (boolean) 내가 쓴 글인지 여부
    - `replies`: (Array) 답장 객체들의 배열
- **Reply Object:**
    - `id`: (string) Unique ID
    - `text`: (string) 답장 내용
    - `timestamp`: (Date) 작성 시간
    - `isAdopted`: (boolean) 채택 여부

### 아키텍처
- **SPA (Single Page Application):** 페이지 이동 없이 동적으로 컨텐츠를 교체합니다.
- **페이지 라우팅:** 간단한 해시 기반 라우팅 또는 JavaScript를 통한 컨테이너 교체 방식으로 "홈", "내 고민", "커뮤니티" 뷰를 전환합니다.
- **데이터 관리:** `localStorage`에서 데이터를 읽고 쓰는 `DataManager` 모듈을 구현하여 데이터 흐름을 관리합니다.

## 4. 개발 계획 (Phase 2)

### 1단계: 구조 리팩토링 및 데이터 관리 설정
- **HTML (`index.html`):**
    - "홈", "내 고민", "커뮤니티" 뷰를 전환할 네비게이션 바 추가
    - 각 뷰를 담을 컨테이너 `<div>` 생성
- **JavaScript (`main.js`):**
    - `localStorage`를 사용한 데이터 CRUD(Create, Read, Update, Delete) 로직 구현
    - 앱 초기화 시 `localStorage`에서 데이터를 불러오고, 없을 경우 초기 더미 데이터(다른 사람의 글 포함)를 생성하는 로직 추가

### 2단계: 신규 기능 페이지 구현
- **"내 고민" 페이지:**
    - 내가 작성한(`isMe: true`) 고민 목록을 렌더링
    - 각 고민에 달린 답장들을 댓글 형태로 표시
    - '채택' 버튼 기능 구현 (클릭 시 `isAdopted` 플래그 변경, 데이터 저장, UI 업데이트)
- **"커뮤니티" 페이지:**
    - 다른 사람이 작성한(`isMe: false`) 고민 목록을 렌더링
    - 각 고민에 답장을 작성할 수 있는 입력 필드와 '답장 남기기' 버튼 추가
    - 답장 제출 시 새로운 답장 객체를 생성하여 해당 고민의 `replies` 배열에 추가하고 데이터 저장, UI 업데이트
- **CSS (`style.css`):**
    - 네비게이션 바 스타일링
    - '채택'된 답장을 시각적으로 구분할 수 있는 스타일 추가 (예: 왕관 아이콘, 하이라이트 배경색)
    - 커뮤니티 페이지의 답장 입력 폼 스타일링

### 3단계: 기존 기능과 통합
- **"홈" 페이지 (고민 작성):** 고민 제출 시 생성되는 `Worry Object`에 `isMe: true` 플래그를 추가하여 `localStorage`에 저장하도록 로직 수정
- **Web Component (`WorryCard`):** 다양한 뷰(내 고민, 커뮤니티)에서 재사용될 수 있도록 구조를 유연하게 변경하고, '채택' 상태 및 답장 입력 폼 등을 조건부로 렌더링하도록 수정
